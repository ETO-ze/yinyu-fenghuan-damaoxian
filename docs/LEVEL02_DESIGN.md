# 第二关设计：云桥花园 · 浮云试炼

## 关卡定位

第二关不是第一关换皮，而是加入新的节奏型平台机制。

| 项目 | 内容 |
| --- | --- |
| 关卡 ID | level_02 |
| 中文名 | 云桥花园 · 浮云试炼 |
| 英文参考名 | Cloudbridge Garden |
| 通关条件 | 收集至少 6 根羽毛，并抵达终点传送门 |
| 核心机制 | 会闪烁、消失、恢复的云平台 |
| 风格 | 亮蓝天空、白云花园、蓝晶、浮空遗迹 |
| 调性 | 非暴力、轻松、积极、冒险感 |

## 玩法目标

玩家从左侧起点出发，先经过稳定石平台，随后学习云平台机制。第二段云平台组合要求玩家掌握节奏，最终到达右侧终点塔楼传送门。

## 机制：CloudPlatform

状态：

```text
solid：正常可站立
warning：闪烁提示即将消失
hidden：隐藏并关闭碰撞
```

时间建议：

```text
踩上后 0.8 秒进入 warning
踩上后 1.4 秒进入 hidden
hidden 3.0 秒后恢复
```

## 关卡结构

```text
起点稳定平台
→ 第一组云平台教学
→ 中段休息平台
→ 第二组云平台进阶
→ 终点高塔传送门
```

## 节奏要求

第一组云平台：

```text
cloud_a → cloud_b → cloud_c
```

目标：让玩家理解云平台会消失，但难度较低。

第二组云平台：

```text
cloud_d → cloud_e → cloud_f → tower_step_a
```

目标：让玩家利用跳跃、滑翔和疾跑完成节奏通过。

## 检查点

```text
start：起点
garden_mid：中段休息平台
before_tower：终点塔前
```

## 美术方向

先不做精修美术，优先保证机制可玩。后续可补云平台正式贴图、云藤、蓝花、云桥花园专用远景。

## 不做内容

```text
1. 不做敌人。
2. 不做攻击。
3. 不做血腥或伤害表现。
4. 不做复杂剧情。
5. 不做新的完整 UI 系统。
```

## 成功标准

```text
1. 第一关通关后能进入第二关。
2. 第二关有清晰的新机制。
3. 云平台行为稳定。
4. 第二关可以完整通关。
5. 第一关不被破坏。
```
